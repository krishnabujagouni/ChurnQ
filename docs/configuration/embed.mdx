## Overview

ChurnShield works via a lightweight JavaScript snippet (`cs.js`) that intercepts your cancel button client-side. When a subscriber clicks cancel, a chat overlay appears in-place — no redirects, no server changes required.

---

## Step 1 — Add the snippet

Paste this into the `<head>` of every page where your cancel button lives. Get your `data-app-id` from **Dashboard → Integration**.

```html
<script
  src="https://cdn.churnshield.ai/cs.js"
  data-app-id="cs_app_YOUR_APP_ID"
  defer
></script>
```

<Warning>
  Use `defer` — not `async`. The snippet must finish loading before your cancel button renders, otherwise the intercept won't fire.
</Warning>

---

## Step 2 — Identify the subscriber

Before the cancel button is shown, call `ChurnShield.identify()` with the subscriber's details. This must run after the script loads.

```javascript
ChurnShield.identify({
  subscriberId: "cus_abc123",           // Stripe customer ID
  subscriptionId: "sub_xyz789",         // Stripe subscription ID (recommended)
  subscriberEmail: "user@example.com",  // shown in your dashboard
  subscriptionMrr: 49.00,               // monthly value in USD
  authHash: "YOUR_HMAC_HASH",           // HMAC-SHA256 signature — see Signing guide
});
```

| Field | Required | Description |
|-------|----------|-------------|
| `subscriberId` | Yes | Stripe customer ID (`cus_...`) |
| `subscriptionId` | Recommended | Stripe subscription ID (`sub_...`). Without it, ChurnShield uses the customer's first active subscription. |
| `subscriberEmail` | No | Shown in the dashboard instead of the raw customer ID |
| `subscriptionMrr` | Recommended | Monthly value in USD — used to calculate offer amounts and the 15% success fee |
| `authHash` | Required in production | HMAC-SHA256 signature — see [HMAC Signing](/configuration/hmac-signing) |

<Tip>
  `ChurnShield.identify()` is safe to call on every page load. If the same subscriber calls it multiple times, the last call wins.
</Tip>

---

## Step 3 — Mark your cancel button

Add `data-cs-cancel` to your cancel button or link:

```html
<button data-cs-cancel>Cancel my subscription</button>
```

ChurnShield intercepts clicks on any element with `data-cs-cancel`. Works with buttons, links, and any other clickable element.

---

## How the flow works

```
Subscriber clicks [data-cs-cancel] element
        ↓
cs.js intercepts the click
        ↓
ChurnShield opens a chat overlay in-place
        ↓
Aria (AI agent) converses with the subscriber
        ↓
Subscriber accepts offer → offer applied in Stripe → overlay closes
Subscriber declines → overlay closes → your cancel flow continues
```

No server changes or redirects required.

## Full example

```html
<!DOCTYPE html>
<html>
<head>
  <script
    src="https://cdn.churnshield.ai/cs.js"
    data-app-id="cs_app_YOUR_APP_ID"
    defer
  ></script>
</head>
<body>
  <script>
    document.addEventListener("DOMContentLoaded", () => {
      ChurnShield.identify({
        subscriberId: "cus_abc123",
        subscriptionId: "sub_xyz789",
        subscriberEmail: "user@example.com",
        subscriptionMrr: 49.00,
        authHash: "YOUR_HMAC_HASH",
      });
    });
  </script>

  <button data-cs-cancel>Cancel subscription</button>
</body>
</html>
```

## Grace mode vs secured mode

When you first install the snippet it runs in **grace mode** — no `authHash` is required. This lets you test quickly without a backend change.

Before going live, **activate signing** in Dashboard → Integration. Once activated, every cancel intent requires a valid `authHash`. See [HMAC Signing](/configuration/hmac-signing).

<Warning>
  Running in grace mode in production means anyone who knows your `data-app-id` can trigger the retention chat for any subscriber ID. Always activate signing before going live.
</Warning>

## Troubleshooting

<AccordionGroup>
  <Accordion title="The chat doesn't open when I click the cancel button">
    Check that the snippet tag is in the page `<head>` with the correct `data-app-id`, that `ChurnShield.identify()` was called before the click, and that the cancel button has the `data-cs-cancel` attribute. Look for any console errors.
  </Accordion>

  <Accordion title="I see 'invalid_auth_hash' in the console">
    The `authHash` you're passing does not match. Verify you are computing `HMAC-SHA256(CHURNSHIELD_EMBED_SECRET, subscriberId)` on your server using the secret from Dashboard → Integration. See [HMAC Signing](/configuration/hmac-signing).
  </Accordion>

  <Accordion title="Sessions don't appear in the dashboard">
    Verify that `subscriptionMrr` is a number greater than 0 and that `subscriberId` matches the Stripe customer ID (`cus_...`) exactly.
  </Accordion>
</AccordionGroup>
