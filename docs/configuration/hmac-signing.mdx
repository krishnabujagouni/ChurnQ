## Overview

When the ChurnShield snippet calls `ChurnShield.identify()`, it sends the subscriber's ID to ChurnShield's servers. Without signing, anyone who knows your `data-app-id` could pass a fake subscriber ID and trigger the retention chat.

HMAC signing solves this: your server computes a cryptographic signature of the `subscriberId` and passes it as `authHash`. ChurnShield verifies the signature before creating a session. Forged requests are rejected.

<Warning>
  Always activate HMAC signing before going live. Running unsigned in production is a security risk.
</Warning>

---

## How it works

1. Your server computes `HMAC-SHA256(CHURNSHIELD_EMBED_SECRET, subscriberId)` and sends the hex digest to your frontend.
2. Your frontend passes the hex string as `authHash` in `ChurnShield.identify()`.
3. ChurnShield verifies the hash using the same secret. If it doesn't match, the request is rejected with `invalid_auth_hash`.

The `CHURNSHIELD_EMBED_SECRET` is a private key that never leaves your server.

---

## Finding your embed secret

1. Go to **Dashboard → Integration**
2. Your embed secret is shown in the **Step 3 — Secure your embed** section
3. Click **Rotate embed secret** to generate a new one (invalidates all existing hashes)

<Warning>
  Never expose the embed secret in client-side code, environment variables prefixed with `NEXT_PUBLIC_`, or git repositories. Treat it like a password.
</Warning>

## Computing the hash on your server

The hash is `HMAC-SHA256(secret, subscriberId)` encoded as a lowercase hex string (64 characters).

<CodeGroup>
```typescript Node.js / TypeScript
import crypto from "crypto";

function getChurnShieldAuthHash(subscriberId: string): string {
  return crypto
    .createHmac("sha256", process.env.CHURNSHIELD_EMBED_SECRET!)
    .update(subscriberId)
    .digest("hex");
}

// Example: in a Next.js API route or server component
const authHash = getChurnShieldAuthHash(user.stripeCustomerId);
// Pass authHash to the client via props, session, or API response
```

```python Python
import hmac
import hashlib
import os

def get_churnshield_auth_hash(subscriber_id: str) -> str:
    secret = os.environ["CHURNSHIELD_EMBED_SECRET"]
    return hmac.new(
        secret.encode("utf-8"),
        subscriber_id.encode("utf-8"),
        hashlib.sha256,
    ).hexdigest()
```

```ruby Ruby
require "openssl"

def churnshield_auth_hash(subscriber_id)
  OpenSSL::HMAC.hexdigest(
    "SHA256",
    ENV["CHURNSHIELD_EMBED_SECRET"],
    subscriber_id
  )
end
```

```php PHP
function churnshield_auth_hash(string $subscriber_id): string {
    return hash_hmac("sha256", $subscriber_id, $_ENV["CHURNSHIELD_EMBED_SECRET"]);
}
```

```go Go
import (
  "crypto/hmac"
  "crypto/sha256"
  "encoding/hex"
  "os"
)

func ChurnShieldAuthHash(subscriberID string) string {
  secret := os.Getenv("CHURNSHIELD_EMBED_SECRET")
  mac := hmac.New(sha256.New, []byte(secret))
  mac.Write([]byte(subscriberID))
  return hex.EncodeToString(mac.Sum(nil))
}
```
</CodeGroup>

## Passing the hash to the frontend

Compute the hash server-side and pass it to your frontend. Never compute it in the browser.

<AccordionGroup>
  <Accordion title="Next.js — server component or server action">
    ```tsx
    // app/account/page.tsx (server component)
    import crypto from "crypto";

    export default async function AccountPage() {
      const user = await getCurrentUser();
      const authHash = crypto
        .createHmac("sha256", process.env.CHURNSHIELD_EMBED_SECRET!)
        .update(user.stripeCustomerId)
        .digest("hex");

      return <AccountClient authHash={authHash} user={user} />;
    }
    ```

    ```tsx
    // app/account/client.tsx
    "use client";
    import { useEffect } from "react";

    export function AccountClient({ authHash, user }) {
      useEffect(() => {
        window.ChurnShield?.identify({
          subscriberId: user.stripeCustomerId,
          subscriptionId: user.stripeSubscriptionId,
          subscriptionMrr: user.monthlyPlanAmount,
          authHash,
        });
      }, []);

      return <button data-cs-cancel>Cancel subscription</button>;
    }
    ```
  </Accordion>

  <Accordion title="Express — inject into page template">
    ```javascript
    const crypto = require("crypto");

      const authHash = crypto
        .createHmac("sha256", process.env.CHURNSHIELD_EMBED_SECRET)

    ```

## Activating signing in the dashboard

Once your backend is computing and passing the `authHash`, activate signing to enforce it:

1. Go to **Dashboard → Integration**
2. Click **Rotate embed secret** — this generates a new secret and locks the embed
3. Update `CHURNSHIELD_EMBED_SECRET` in your server's environment variables with the new secret

After activation, any request without a valid `authHash` is rejected. Test your integration before activating to avoid disrupting live users.

---

## Rotating the secret

To rotate the secret:
1. Click **Rotate embed secret** in Dashboard → Integration
2. A new secret is generated immediately — the old one is invalidated
3. Update `CHURNSHIELD_EMBED_SECRET` in your environment and redeploy

<Warning>
  Rotating the secret invalidates all previously computed hashes. Make sure your server is using the new secret before old hashes expire from caches.
</Warning>

## Implementation notes

<AccordionGroup>
  <Accordion title="The subscriberId must match exactly">
    The hash is computed over the exact string you pass as `subscriberId`. If you use `cus_abc123` in `identify()` but computed the hash over `CUS_ABC123`, verification will fail. Keep casing consistent.
  </Accordion>

  <Accordion title="Do not compute the hash in the browser">
    If you compute the hash in JavaScript running in the browser, the secret is exposed in your source code or network requests. Always compute it server-side and pass it down as a value.
  </Accordion>

  <Accordion title="Grace mode still works without a hash">
    If signing is not yet activated, requests without an `authHash` are accepted with a warning header (`X-ChurnShield-Warning: embed_unsigned`). Activate signing before going live.
  </Accordion>
</AccordionGroup>
