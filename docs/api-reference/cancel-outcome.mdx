---
title: "GET /api/cancel-outcome"
description: "Retrieve the outcome of a cancellation intent after the subscriber has made a decision."
---

## Endpoint

```
GET https://app.churnshield.ai/api/cancel-outcome/{intentId}
```

## Authentication

```
Authorization: Bearer YOUR_API_KEY
```

---

## When to use this

You typically do not need to poll this endpoint  the `returnUrl` you supplied to `/api/cancel-intent` receives `?cs_result=accepted` or `?cs_result=declined` when the subscriber finishes. Use this endpoint when:

- You need to fetch the outcome server-side (e.g. in a webhook processor).
- You want to retrieve the offer type that was presented and accepted.
- You are building a dashboard or audit log of cancellation outcomes.

---

## Path parameters

| Parameter | Description |
|-----------|-------------|
| `intentId` | The `id` returned by `POST /api/cancel-intent` (format: `ci_...`) |

## Response

### 200 OK  decision made

```json
{
  "id": "ci_abc123xyz",
  "status": "accepted",
  "offerType": "discount",
  "offerDetail": {
    "discountPercent": 20,
    "duration": "3_months",
    "stripeCouponId": "CHURN20_3M"
  },
  "customerId": "cus_Abc123",
  "subscriptionId": "sub_Xyz789",
  "cancelReason": "too_expensive",
  "decidedAt": "2026-04-06T14:35:22.000Z",
  "createdAt": "2026-04-06T14:22:01.000Z"
}
```

### 200 OK  pending (subscriber has not decided yet)

```json
{
  "id": "ci_abc123xyz",
  "status": "pending",
  "customerId": "cus_Abc123",
  "subscriptionId": "sub_Xyz789",
  "createdAt": "2026-04-06T14:22:01.000Z"
}
```

### 200 OK  expired

```json
{
  "id": "ci_abc123xyz",
  "status": "expired",
  "customerId": "cus_Abc123",
  "subscriptionId": "sub_Xyz789",
  "createdAt": "2026-04-06T14:22:01.000Z"
}
```

---

## Response fields

| Field | Type | Description |
|-------|------|-------------|
| `id` | `string` | Cancel intent ID |
| `status` | `string` | `pending`, `accepted`, `declined`, or `expired` |
| `offerType` | `string` \| `null` | `discount`, `pause`, `downgrade`, `free_month`, or `custom`. `null` if status is not `accepted` |
| `offerDetail` | `object` \| `null` | Details of the accepted offer. Shape depends on `offerType` |
| `customerId` | `string` | Stripe customer ID |
| `subscriptionId` | `string` | Stripe subscription ID |
| `cancelReason` | `string` \| `null` | Reason the subscriber gave for cancelling |
| `decidedAt` | `string` \| `null` | ISO 8601 timestamp when the subscriber made their decision |
| `createdAt` | `string` | ISO 8601 timestamp when the intent was created |

### `offerDetail` shapes

**Discount:**
```json
{
  "discountPercent": 20,
  "duration": "3_months",
  "stripeCouponId": "CHURN20_3M"
}
```

**Pause:**
```json
{
  "pauseMonths": 2,
  "resumeAt": "2026-06-06T00:00:00.000Z"
}
```

**Downgrade:**
```json
{
  "fromPriceId": "price_PremiumMonthly",
  "toPriceId": "price_StarterMonthly"
}
```

**Free month:**
```json
{
  "billingCycleWaived": 1,
  "stripeCouponId": "CHURN_FREE1M"
}
```

---

## Error responses

| Status | Code | Meaning |
|--------|------|---------|
| `401` | `unauthorized` | Missing or invalid API key |
| `404` | `not_found` | No intent with this ID exists in your tenant |

## Example

<CodeGroup>
```bash cURL
curl https://app.churnshield.ai/api/cancel-outcome/ci_abc123xyz \
  -H "Authorization: Bearer YOUR_API_KEY"
```

```typescript TypeScript
const res = await fetch(
  `https://app.churnshield.ai/api/cancel-outcome/${intentId}`,
  {
    headers: {
      "Authorization": `Bearer ${process.env.CHURNSHIELD_API_KEY}`,
    },
  }
);

const outcome = await res.json();

if (outcome.status === "accepted") {
  console.log(`Offer accepted: ${outcome.offerType}`);
} else if (outcome.status === "declined") {
  console.log("Subscriber declined  subscription cancelled");
}
```

```python Python
import httpx

res = httpx.get(
    f"https://app.churnshield.ai/api/cancel-outcome/{intent_id}",
    headers={"Authorization": f"Bearer {CHURNSHIELD_API_KEY}"},
)
res.raise_for_status()
outcome = res.json()

if outcome["status"] == "accepted":
    print(f"Offer accepted: {outcome['offerType']}")
elif outcome["status"] == "declined":
    print("Subscriber declined  subscription cancelled")
```
</CodeGroup>

## Webhooks (preferred alternative)

Instead of polling this endpoint, subscribe to `cancel_intent.accepted` and `cancel_intent.declined` webhook events. Webhooks are pushed instantly when the decision is made and are more efficient than polling.

See [Webhooks](/integrations/webhooks) for setup instructions.
